1. Comments

Comments in JavaScript are used to add explanatory text within the code. They are ignored by the JavaScript interpreter and do not affect the execution of the program. Comments can be used to improve code readability, explain functionality, or temporarily disable code.

Syntax: 


//  a single-line comment

/*
     a multi-line comment.
     can span multiple lines.
*/

2. Variables

Variables are used to store and access the data in JavaScript. They provide a way to give a name (identifier) to a value, which can be a number, string, boolean, object, or any other data type.

Syntax: 


var variableName = value;  // Declaring a variable (old way, not recommended)

let variableName = value;  // Declaring a variable (block-scoped)

const variableName = value;  // Declaring a constant (block-scoped)

Example: 


let name = "John";
let age = 25;
let isStudent = true;

console.log(name);       // Output: John
console.log(age);        // Output: 25
console.log(isStudent);  // Output: true

name = "Alice";
age = 30;
isStudent = false;

console.log(name);       // Output: Alice
console.log(age);        // Output: 30
console.log(isStudent);  // Output: false

name = 30  // dynamic : re-assining a different value to other data type variable
console.log(name) // 30

3. Let and Const

let and const are keywords introduced in ES6 (ECMAScript 2015) for declaring variables. They have block scope, meaning they are only accessible within the block (enclosed within curly braces) where they are defined.

let allows reassigning a new value to the variable, whereas const creates a read-only constant that cannot be reassigned once initialized.

Example: 


let a = 5;
a = 10;
console.log(a);  // Output: 10

const b = 15;
console.log(b) // 15
b = 20;       
console.log(b);  // ERROR!

4. Data Types

JavaScript has several built-in data types

Example: 


Number: numeric values, such as integers and floating-point numbers.
String: series of characters like text, sentences, etc.
Boolean: a logical value, true or false.
Object: a collection of key-value pairs or properties.
Array: an ordered list of values.
Null: the absence of any object value.

6. if

The if statement is used to execute a block of code only if a specified condition is true.

Syntax: 


if (condition) {
     // Code block if the condition is true
}

Example: 


let age = 20;

if (age >= 18) {
     console.log("You are an adult.");
}

7. else if

The else if statement allows you to check additional conditions if the preceding if statement is false.

Syntax: 


if (condition1) {
     // Code block  if condition1 is true
} else if (condition2) {
     // Code block if condition2 is true
}

Example: 


let time = 14;

if (time < 12) {
     console.log("Good morning!");
} else if (time  < 18) {
     console.log("Good afternoon!");
}

8. else

The else statement is used to define a block of code to be executed if none of the preceding conditions are true.

Syntax: 


if (condition) {
     // Code block if the condition is true
} else {
     // Code block if the condition is false
}

Example: 


let day = "Monday";

if (day === "Saturday" || day === "Sunday") {
     console.log("It's the weekend!");
} else {
     console.log("It's a weekday.");
}

9. for

The for loop is used to execute a block of code repeatedly for a specified number of times.

Syntax: 


for (initialization; condition; iteration) {
     // Code to be executed in each iteration
}

Example: 


for (let i = 0; i < 5; i++) {
     console.log(i);
}
// Output: 0, 1, 2, 3, 4

10. while

The while loop is used to execute a block of code repeatedly as long as a specified condition is true.

Syntax: 


while (condition) {
     // Code to be executed as long as the condition is true
}

Example: 


let i = 0;
while (i < 5) {
     console.log(i);
     i++;
}
// Output: 0, 1, 2, 3, 4

11. do-while

The code block is executed at least once, regardless of whether the condition is initially false.

Syntax: 


do {
     // Code to be executed
} while (condition);

Example: 


let i = 3;
do {
     console.log(i);
     i--;
} while (i > 5);
console.log("After loop");

// 3
// After loop

12. Switch Statement

The switch statement is used to perform different actions based on different conditions.

Syntax: 


switch (expression) {
case value1:
     // Code block
     break;
case value2:
     // Code block
     break;
default:
     // default code block
}

Example: 


let day = "Monday";
switch (day) {
     case "Monday":
     console.log("It's Monday!");
     break;
     case "Tuesday":
     console.log("It's Tuesday!");
     break;
     default:
     console.log("It's another day.");
}
// Output: It's Monday!

13. Break and Continue

The break statement is used to exit a loop or switch statement, while the continue statement is used to skip the current iteration of a loop and continue to the next iteration.

Example: 


// break 
for (let i = 0; i < 5; i++) {
if (i === 3) {
     break;
}
console.log(i);
}
// Output: 0, 1, 2



// continue
for (let i = 0; i < 5; i++) {
if (i === 2) {
     continue;
}
console.log(i);
}
// Output: 0, 1, 3, 4

14. Functions

Functions are blocks of code that can be reused to perform a specific task.

Syntax: 


function functionName(parameter1, parameter2) {
     // Code to be executed
     return result; // Optional
}
  
functionName(argument1, argument2); // function call

Example: 


function hi(name) {
     console.log("Hi, " + name + "!");
}

hi("Radha");
hi("Krihna")

// Hi, Radha!
// Hi, Krihna!

15. Arrow Functions

Arrow functions are a concise syntax for writing functions in JavaScript.

Syntax: 


const functionName = (parameter1, parameter2) => {
     // Code to be executed
     return result; // Optional
}

functionName(argument1, argument2); // function call

Example: 


const hi = (name) => { 
     console.log("Hi, " + name + "!"); 
} 

hi("Radha");
hi("Krihna")

// Hi, Radha!
// Hi, Krihna!

16. Arrays

Arrays are utilized to store multiple values within a single variable.

Syntax: 

let arrayName = [value1, value2, value3];

Example: 


let fruits = ["apple", "banana", "orange"];
console.log(fruits);  // Output: ["apple", "banana", "orange"]

17. Array Methods

Array methods are built-in functions that can be used to manipulate arrays.

Example: 


let numbers = [1, 2, 3, 4, 5];

// Push: add an element to the end of the array
numbers.push(6);
console.log(numbers);  // Output: [1, 2, 3, 4, 5, 6]

// Pop: delete the last element from the array
numbers.pop();
console.log(numbers);  // Output: [1, 2, 3, 4, 5]

// Length: get the length of the array
console.log(numbers.length);  // Output: 5

// Concat: merge two arrays
let moreNumbers = [6, 7, 8];
let mergedArray = numbers.concat(moreNumbers);
console.log(mergedArray);  // Output: [1, 2, 3, 4, 5, 6, 7, 8]

// Join: convert the array into a string
let arrayAsString = numbers.join(", ");
console.log(arrayAsString);  // Output: "1, 2, 3, 4, 5"

18. for of

The for...of loop is used to iterate over elements of an iterable object, such as an array.

Syntax: 


for (let element of iterable) {
     // Code to be executed for each element
}

Example: 


let fruits = ["apple", "banana", "orange"];
for (let fruit of fruits) {
     console.log(fruit);
}
// Output: "apple", "banana", "orange"

19. Objects

Objects are used to store key-value pairs and represent complex data structures.

Syntax: 


let objectName = {
     key1: value1,
     key2: value2,
     key3: value3
};

Example: 


let person = {
     name: "John",
     age: 30,
     city: "New York"
};
console.log(person);  // Output: { name: "John", age: 30, city: "New York" }

20. for in

The for...in loop is used to iterate across the properties of an object.

Syntax: 


for (let property in object) {
     // Code to be executed for each property
}

Example: 


let person = {
     name: "Radha",
     age: 30,
     city: "Vizag"
};

for (let key in person) {
console.log(key + ": " + person[key]);
}
// Output: "name: Radha", "age: 30", "city: Vizag"

21. Creating JSON Objects

JSON (JavaScript Object Notation) is a lightweight data interchange format that is designed to be easily readable and writable for humans, as well as easily parsed and generated by machines.

Example: 


let person = {
"name": "John Doe",
"age": 30,
"city": "New York"
};

22. Parsing JSON

Parsing JSON refers to converting a JSON string into a JavaScript object.

Example: 


let jsonString = '{"name":"John Doe","age":30,"city":"New York"}';
let person = JSON.parse(jsonString);
console.log(person.name);  // Output: "John Doe"

23. Stringifying JSON

Stringifying JSON refers to converting a JavaScript object into a JSON string.

Example: 


let person = {
     "name": "John Doe",
     "age": 30,
     "city": "New York"
};
let jsonString = JSON.stringify(person);
console.log(jsonString);  // Output: '{"name":"John Doe","age":30,"city":"New York"}'

27. setTimeout() and setInterval()

The setTimeout() function is used to run a function or evaluate an expression after a specific delay, measured in milliseconds.

The setInterval() function is used to execute a function or evaluate an expression repeatedly at a specified interval (in milliseconds).

Syntax: 


setTimeout(function, delay);
setInterval(function, interval);

Example: 


setTimeout(() => {
     console.log("Delayed execution");
     }, 2000);
     // Output: "Delayed execution" (after 2 seconds)
     
        let count = 0;
let intervalId = setInterval(() => {
  console.log(count);
  count++;
  if (count > 5) {
    clearInterval(intervalId);
  }
}, 1000);
// Output: 0, 1, 2, 3, 4, 5 (printed every second)

28. Callback Functions

A callback function is a function that is passed as an argument to another function and is executed after a certain task is completed.

Example: 


function greet(name, callback) {
     console.log("Hello, " + name + "!");
     callback();
}

function sayGoodbye() {
     console.log("Goodbye!");
}

greet("John", sayGoodbye);
// Output: "Hello, John!", "Goodbye!"

29. Promises

Promises are used for asynchronous operations in JavaScript. They represent the eventual completion (or failure) of an asynchronous operation and allow handling the results.

Example: 


function getData() {
     return new Promise((resolve, reject) => {
          setTimeout(() => {
          const data = "Data received";
          if (data) {
               resolve(data);
          } else {
               reject("Error occurred");
          }
          }, 2000);
     });
     }
     
getData()
.then((data) => {
     console.log(data);
})
.catch((error) => {
     console.log(error);
});
// Output: "Data received" (after 2 seconds)

30. Async/Await

Async/await is a syntax feature that simplifies working with Promises by providing a more intuitive and concise way to handle asynchronous operations.

Example: 


function getData() {
return new Promise((resolve, reject) => {
     setTimeout(() => {
     const data = "Data received";
     if (data) {
          resolve(data);
     } else {
          reject("Error occurred");
     }
     }, 2000);
});
}

async function fetchData() {
try {
     const data = await getData();
     console.log(data);
} catch (error) {
     console.log(error);
}
}

fetchData();
// Output: "Data received" (after 2 seconds)

31. try-catch Blocks

The try-catch block is used for error handling in JavaScript. It allows catching and handling exceptions that may occur within a block of code.

Syntax: 


try {
     //  may throw an exception
} catch (error) {
     //   handle the exception
}

Example: 


try {
const result = 10 / 0;
console.log(result);
} catch (error) {
console.log("An error occurred: " + error);
}
// Output: "An error occurred: Infinity"

32. Throwing Errors

You can manually throw an error using the throw statement. It allows you to specify custom error messages or trigger specific types of errors.

Example: 


function divide(a, b) {
if (b === 0) {
     throw new Error("Can't divide with 0");
}
return a / b;
}

try {
const result = divide(10, 0);
console.log(result);
} catch (error) {
console.log("An error occurred: " + error.message);
}
// Output: "An error occurred: Can't divide with 0"

33. Handling Exceptions

Exception handling refers to the process of catching and handling errors to prevent the program from crashing. It allows you to gracefully handle errors and provide fallback behavior.

Example: 


function fetchData() {
try {
     // Code that may throw an exception
     const data = fetch("https://api.example.com/data");
     return data;
} catch (error) {
     // Code to handle the exception
     console.log("An error occurred: " + error);
     return null;
}
}

const result = fetchData();
if (result) {
// Code to handle the successful result
} else {
// Code to handle the exception case
}

34. Creating Modules

Modules are used to organize and encapsulate code into separate files, making it easier to manage and reuse code.

Example: 


// module.js
export function greet(name) {
     console.log(`Hello, ${name}!`);
}

export function sayGoodbye() {
     console.log("Goodbye!");
}

35. Importing Modules

Exporting modules allows specific functions, variables, or objects to be accessible outside the module. Importing modules allows using those exported entities in other modules.

Example: 


import { greet, sayGoodbye } from './module.js';

greet("John");
// Output: "Hello, John!"

sayGoodbye();
// Output: "Goodbye!"

36. Template Literals

Template literals are a way to concatenate strings and include variables or expressions using backticks (`).

Example: 


let name = "John";
let message = `Hello, ${name}!`;
console.log(message);
// Output: "Hello, John!"

37. Creating Regular Expressions

Regular expressions (regex) are patterns used to match and manipulate text.

Syntax: 


let regex = /pattern/;

Example: 


// Regular expression to match email addresses
let emailRegex = /^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$/;

let email = "sarah@example.com";

if (emailRegex.test(email)) {
     console.log("Valid email address");
} else {
     console.log("Invalid email address");
}
// Output: "Valid email address"

fetch('https://api.example.com/data')

39. Geolocation API

The Geolocation API allows obtaining the geographical location information of the device.

Example: 


if (navigator.geolocation) {
navigator.geolocation.getCurrentPosition(position => {
     console.log('Latitude:', position.coords.latitude);

40. Web Storage API

The Web Storage API provides a way to store data in the browser's storage, including Local Storage, which persists data even after the browser is closed.

Example: 


localStorage.setItem('name', 'John');
let name = localStorage.getItem('name');
console.log(name);  // Output: "John"

localStorage.removeItem('name');

41. Local Storage

Local Storage is a feature in web browsers that allows JavaScript to store data locally on the user's device. It provides a simple key-value storage mechanism and can be used to persist data even when the browser is closed or the page is refreshed.

Example: 


// Storing data in Local Storage
localStorage.setItem('name', 'John');
localStorage.setItem('age', '30');

// Retrieving data from Local Storage
let name = localStorage.getItem('name');
let age = localStorage.getItem('age');

console.log(name); // Output: "John"
console.log(age); // Output: "30"

// Updating data in Local Storage
localStorage.setItem('age', '31');
age = localStorage.getItem('age');
console.log(age); // Output: "31"

// Removing data from Local Storage
localStorage.removeItem('name');
name = localStorage.getItem('name');
console.log(name); // Output: null

// Clearing all data from Local Storage
localStorage.clear();
