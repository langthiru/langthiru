6. Conditional Statements (if, else, else if)

Conditional statements allow you to execute different blocks of code based on specified conditions.

The if statement is used to execute a block of code if a certain condition is true.

The else statement is used in conjunction with the if statement and executes a block of code if the if condition is false.

Syntax: 


if (condition) {
     // code if the condition is true
} else if (condition) {
     // code block if the condition is true
} else {
     // code block if none of the conditions are true
}

Example: 


int a = 10;

if (a > 0) {
     System.out.println("a is positive");
} else if (a < 0) {
     System.out.println("a is negative");
} else {
     System.out.println("a is zero");
}

// Output: a is positive

7. Loops (for, while, do-while)

The for loop allows you to execute a block of code a fixed number of times.

The while loop executes a specific block of code repeatedly until a certain condition remains true.

The do-while loop is similar to the while loop, but it guarantees that the code inside the loop is executed at least once, as the condition is evaluated after the loop body.

Syntax: 


for (initialization; condition; update) {
     // code to be executed
}

while (condition) {
     // code to be executed
}

do {
     // code to be executed
} while (condition);

Example: 


// for loop

    a--;
}

// do-while loop
System.out.println("do-while loop");
int j = 1;
do {
    System.out.println("Count: " + j);

8. Switch Statement

The switch statement allows you to select one of many code blocks to be executed based on a specified expression.

Syntax: 


switch (expression) {
case value1:
     // code block if the expression matches value1
     break;
case value2:
     // code block  if the expression matches value2
     break;
...
default:
     // code tblock if none of the cases match the expression
}

Example: 


int day = 3;
String dayName;

switch (day) {
     case 1:
          dayName = "Monday";
          break;
     case 2:
          dayName = "Tuesday";
          break;
     case 3:
          dayName = "Wednesday";
          break;
     case 4:
          dayName = "Thursday";
          break;
     case 5:
          dayName = "Friday";
          break;
     default:
          dayName = "Invalid day";
          break;
}

System.out.println("Day: " + dayName); // Day: Wednesday

9. Break and Continue statements

The break statement is used to terminate the execution of a loop or switch statement, while the continue statement is used to skip the remaining code inside a loop and move to the next iteration.

Syntax: 


// break syntax
break;

// continue syntax
continue;

Example: 


for (int i = 1; i <= 10; i++) {
     if (i == 5) {
          break; // terminates the loop when i is 5
     }
     System.out.println("Count: " + i);
}

for (int i = 1; i <= 10; i++) {
     if (i % 2 == 0) {
          continue; // skips the remaining code and moves to the next iteration for even numbers
     }
     System.out.println("Count: " + i);
}

10. Arrays

An array is a data structure that allows you to store multiple values of the same type in a single variable. It provides an indexed collection of elements, where each element can be accessed using its index.

Syntax: 


// Declaration syntax
dataType[] arrayName; // preferred syntax
or
dataType arrayName[]; // alternative syntax

// Initialization syntax
dataType[] arrayName = new dataType[arrayLength];
or
dataType[] arrayName = {element1, element2, ..., elementN};

Example: 


int[] numbers = new int[5]; // declares an array of integers with length 5

numbers[0] = 10; // assigns 10 to the first element
numbers[1] = 20; // assigns 20 to the second element
// ...
numbers[4] = 50; // assigns 50 to the fifth element

String[] names = {"Alice", "Bob", "Charlie"}; // declares and initializes an array of strings

11. Classes and Objects

A class is a blueprint for creating objects. It defines the properties (attributes) and behaviors (methods) that objects of that class can have. An object is an instance of a class, which represents a specific entity or concept.

Syntax: 


// Class Declaration
class ClassName {
     // instance variables (attributes)
     // constructors
     // methods
}

// Object Creation
ClassName objectName = new ClassName();

Example: 


class Car {
     String color;
     int year;

     void startEngine() {
          System.out.println("Engine started.");
     }

     void accelerate() {
          System.out.println("Car is accelerating.");
     }
     }

     public class Main {
     public static void main(String[] args) {
          Car myCar = new Car(); // creates an instance of the Car class
          myCar.color = "Red"; // sets the color of the car
          myCar.year = 2021; // sets the year of the car
          myCar.startEngine(); // invokes the startEngine() method
     }
}

12. Constructors

A constructor is a unique method employed for initializing objects within a class. It is automatically invoked when an object is created. Constructors share the same name as the class and do not specify a return type.

Syntax: 


ClassName() {
     // constructor code
}

Example: 


class Person {
     String name;
     int age;

     Person() {
          name = "John";
          age = 30;
     }

     void displayInfo() {
          System.out.println("Name: " + name);
          System.out.println("Age: " + age);
     }
}

Person person = new Person(); // creates an instance of the Person class and invokes the constructor
person.displayInfo(); // invokes the displayInfo() method

13. Instance Variables and Methods

Instance variables are variables declared inside a class but outside any method. They hold the state or data of an object. Instance methods are methods defined in a class that can access and manipulate the instance variables of an object.

Syntax: 


// Instance Variable Declaration
accessModifier dataType variableName;

// Instance Method Declaration
accessModifier returnType methodName(parameters) {
     // method code
 }

Example: 


class Circle {
     double radius; // instance variable

     double calculateArea() { // instance method
          return 3.14 * radius * radius;
     }
}

Circle myCircle = new Circle();
myCircle.radius = 5.0;
double area = myCircle.calculateArea();
System.out.println("Area: " + area);

14. Access Modifiers (public, private, protected)

Access modifiers determine the visibility or accessibility of classes, methods, and variables in Java.

public: The public access modifier allows classes, methods, and variables to be accessed from anywhere, both within and outside the class or package.

private: The private access modifier restricts the access to the class itself. Private methods and variables cannot be accessed from outside the class.

protected: The protected access modifier allows access within the same class, package, and subclasses. Protected methods and variables can be accessed within the class, subclasses, and same package.

15. Inheritance

Inheritance is a mechanism in Java that allows a class (called the subclass or derived class) to inherit the properties and behaviors of another class (called the superclass or base class). The subclass can extend the functionality of the superclass and add its own additional features.

Syntax: 


class Subclass extends Superclass {
     // subclass members
}

Example: 


class Animal { // superclass
     void eat() {
          System.out.println("Animal is eating.");
     }
}

class Dog extends Animal { // subclass
     void bark() {
          System.out.println("Dog is barking.");
     }
}

public class Main {
public static void main(String[] args) {
     Dog dog = new Dog();
     dog.eat(); // inherited from Animal class
     dog.bark();
}
}

16. Interfaces

In Java, an interface is like a promise that classes make to follow certain rules. It tells them what methods they need to have. It specifies a set of methods that implementing classes must define. An interface cannot be instantiated; it only provides a way to define a common behavior for multiple classes.

Syntax: 


// Interface Declaration
interface InterfaceName {
     // method declarations
}
    
// Implementing an Interface
class ClassName implements InterfaceName {
     // class members
 }

Example: 


interface Vehicle {
void start();
void stop();
}

class Car implements Vehicle {
     public void start() {
          System.out.println("Car started.");
     }

     public void stop() {
          System.out.println("Car stopped.");
     }
}

public class Main {
public static void main(String[] args) {
     Car car = new Car();
     car.start();
     car.stop();
}
}

17. Try-Catch Blocks

Try-catch blocks are used to handle exceptions (errors) that may occur during the execution of a program. The code inside the try block is monitored for exceptions, and if an exception occurs, it is caught and processed in the catch block.

Syntax: 


try {
     // code that may throw an exception
} catch (ExceptionType exceptionObject) {
     // code to handle the exception
}

Example: 


public class Main {
public static void main(String[] args) {
int[] numbers = {1, 2, 3, 4, 5};

     try {
          System.out.println(numbers[10]); // accessing an out-of-bounds index
     } catch (ArrayIndexOutOfBoundsException e) {
          System.out.println("Array index out of bounds exception caught.");
     }
}
}

18. Multiple Catch Blocks

Multiple catch blocks can be used to handle different types of exceptions separately. Each catch block can catch a specific type of exception, allowing for customized error handling based on the type of exception.

Syntax: 


try {
     // code that may throw an exception
} catch (ExceptionType1 exceptionObject1) {
     // code to handle exceptionType1
} catch (ExceptionType2 exceptionObject2) {
     // code to handle exceptionType2
} // more catch blocks if needed

Example: 


public class Main {
public static void main(String[] args) {
     try {
          int[] numbers = {1, 2, 3};
          System.out.println(numbers[10]); // accessing an out-of-bounds index
     } catch (ArrayIndexOutOfBoundsException e) {
          System.out.println("Array index out of bounds exception caught.");
     } catch (NullPointerException e) {
          System.out.println("Null pointer exception caught.");
     }
}
}

19. Finally Block

The finally block works together with the try-catch block to define a code section that will always run, regardless of whether an exception is encountered or not. The code inside the finally block is guaranteed to be executed, regardless of any exceptions that may arise.

Syntax: 


try {
     // code that may throw an exception
} catch (ExceptionType exceptionObject) {
     // code to handle the exception
} finally {
     // code to be executed regardless of an exception
}

Example: 


public class Main {
public static void main(String[] args) {
     try {
          int result = 10 / 0; // division by zero
          System.out.println("Result: " + result); // this line will not be executed
     } catch (ArithmeticException e) {
          System.out.println("Exception caught: Division by zero.");
     } finally {
          System.out.println("Finally block executed.");
     }
}
}

20. Custom Exception

A custom exception, also known as a user-defined exception, allows you to create your own exception classes by extending the existing exception classes provided by Java. Custom exceptions can be used to handle specific scenarios or errors in your program.

Syntax: 


class CustomException extends Exception {
     // constructors, methods, etc.
}

Example: 


class InsufficientBalanceException extends Exception {
InsufficientBalanceException(String message) {
     super(message);
}
}

class BankAccount {
private double balance;

void withdraw(double amount) throws InsufficientBalanceException {
     if (amount > balance) {
          throw new InsufficientBalanceException("Insufficient balance.");
     }
     balance -= amount;
     System.out.println("Withdrawal successful. Remaining balance: " + balance);
}
}

public class Main {
public static void main(String[] args) {
     BankAccount account = new BankAccount();
     account.deposit(1000);
     try {
          account.withdraw(1500); // throws InsufficientBalanceException
     } catch (InsufficientBalanceException e) {
          System.out.println(e.getMessage());
     }
}
}

22. File Handling

File handling in Java allows you to read from and write to files on your system. The java.io package provides classes like File, FileReader, FileWriter, BufferedReader, and BufferedWriter to handle file operations.

Example: 


// Reading from a File
import java.io.File;
import java.io.FileReader;
import java.io.IOException;

public class Main {
     public static void main(String[] args) {
          try {
               File file = new File("example.txt");
               FileReader reader = new FileReader(file);
               int character;
               while ((character = reader.read()) != -1) {
                    System.out.print((char) character);
               }
               reader.close();
          } catch (IOException e) {
               System.out.println("Error reading file: " + e.getMessage());
          }
     }
}
     
// Writing to a File
import java.io.File;
import java.io.FileWriter;
import java.io.IOException;

public class Main {
    public static void main(String[] args) {
        try {
            File file = new File("example.txt");
            FileWriter writer = new FileWriter(file);
            writer.write("Hello, World!");
            writer.close();
            System.out.println("Data written to file.");
        } catch (IOException e) {
            System.out.println("Error writing to file: " + e.getMessage());
        }
    }
}

25. Thread Pools

Thread pools provide a way to manage and reuse a pool of worker threads. Instead of creating a new thread for every task, a thread pool allows you to submit tasks to the pool, which assigns the tasks to available threads. This improves performance by reducing thread creation overhead.

Example: 


import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class Main {
public static void main(String[] args) {
     ExecutorService executor = Executors.newFixedThreadPool(5);

     // Submit tasks to the thread pool
     for (int i = 0; i < 10; i++) {
          Runnable task = new MyTask();
          executor.execute(task);
     }

     executor.shutdown(); // Shutdown the thread pool
}
}

class MyTask implements Runnable {
public void run() {
     // code to be executed in the task
}
}

26. Thread Communication

Thread communication allows threads to synchronize their actions and exchange data. The wait(), notify(), and notifyAll() methods of the Object class are used for thread communication.

Example: 


class Data {
private String message;
private boolean isProduced = false;

public synchronized void produce(String message) {
     while (isProduced) {
          try {
               wait(); // wait until consumer consumes the previous message
          } catch (InterruptedException e) {
               e.printStackTrace();
          }
     }
     this.message = message;
     isProduced = true;
     notify(); // notify the consumer that a new message is available
}

public synchronized String consume() {
     while (!isProduced) {
          try {
               wait(); // wait until producer produces a message
          } catch (InterruptedException e) {
               e.printStackTrace();
          }
     }
     isProduced = false;
     notify(); // notify the producer that the message has been consumed
     return message;
}
}

public class Main {
public static void main(String[] args) {
     Data data = new Data();

     Thread producerThread = new Thread(() -> {
          for (int i = 0; i < 5; i++) {
               String message = "Message " + (i + 1);
               data.produce(message);
               System.out.println("Produced: " + message);
          }
     });

     Thread consumerThread = new Thread(() -> {
          for (int i = 0; i < 5; i++) {
               String message = data.consume();
               System.out.println("Consumed: " + message);
          }
     });

     producerThread.start();
     consumerThread.start();
}
}
